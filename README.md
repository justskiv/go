# April Fools' Prank: Go Inheritance Proposal

A pixel-perfect replica of a GitHub issue page for the Go repository, proposing to add classical inheritance to Go.

Built as an April 1st prank for the Go community.

## What is this?

A static HTML page that looks exactly like a real GitHub issue in [golang/go](https://github.com/golang/go), complete with:

- Dark theme matching GitHub's current design
- Realistic proposal text with code examples and syntax highlighting
- Comments from "community members" pushing back on the idea
- Working UI elements — dropdowns, copy buttons, search field
- Real GitHub avatars and profile links
- Open Graph tags for convincing Telegram/social media previews

## The "Proposal"

> **proposal: spec: add classical inheritance to Go**
>
> Introduces `class` and `extends` keywords, single-inheritance vtable dispatch, and true subtyping.
> Prototype passes 97.3% of stdlib tests.

The link to the "prototype repository" leads to a Rick Astley video.

## Live

Hosted via GitHub Pages: **https://justskiv.github.io/go/issues/78847**

## Disclaimer

This is a joke. Classical inheritance has no place in Go, and everyone involved in the Go project has been very clear about that since before Go 1.0.

No real GitHub issues were harmed in the making of this prank.
